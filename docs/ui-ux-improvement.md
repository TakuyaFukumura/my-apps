# UI/UX 改善提案

## 概要

一覧画面のユーザビリティ向上を目的とした UI/UX 改善の仕様をまとめます。

---

## 1. アプリ名クリックで詳細画面へ遷移

### 現状

- 一覧画面でアプリ名（`<h2>`）はテキスト表示のみで、クリックしても何も起こらない
- 詳細画面へ遷移するためには「詳細を見る →」リンクをクリックする必要がある

### 改善内容

- アプリ名をリンク（`<Link>`）に変更し、クリックすると詳細画面（`/apps/:id`）へ遷移する
- アプリ名にホバー時のスタイル（下線・色変化など）を付与し、クリック可能であることをユーザーに伝える

### 変更対象ファイル

- `src/app/page.tsx`

### 変更前後のイメージ

**変更前**

```tsx
<h2 className="text-lg font-bold text-gray-800 dark:text-gray-200 mb-3">
    {app.name}
</h2>
```

**変更後**

```tsx
<h2 className="text-lg font-bold mb-3">
    <Link
        href={`/apps/${app.id}`}
        className="text-gray-800 dark:text-gray-200 hover:text-blue-600 dark:hover:text-blue-400 hover:underline transition-colors duration-200"
    >
        {app.name}
    </Link>
</h2>
```

## 2. 「詳細を見る →」リンクの削除

### 現状

- 一覧画面の各カードに「詳細を見る →」リンクが表示されている
- アプリ名クリックで詳細画面へ遷移できるようになれば、このリンクは冗長になる

### 改善内容

- 「詳細を見る →」リンクを削除し、カードのレイアウトをすっきりさせる

### 変更対象ファイル

- `src/app/page.tsx`

### 変更前後のイメージ

**変更前**

```tsx
<div className="flex flex-col gap-2 mt-auto">
    {app.siteUrl && (
        <a href={app.siteUrl} ...>
            サイトを見る
        </a>
    )}
    <Link href={`/apps/${app.id}`} ...>
        詳細を見る →
    </Link>
</div>
```

**変更後**

```tsx
<div className="flex flex-col gap-2 mt-auto">
    {app.siteUrl && (
        <a href={app.siteUrl} ...>
            サイトを見る
        </a>
    )}
</div>
```

## 3. その他の UI/UX 改善提案

### 3-1. カード全体をクリック可能にする

#### 現状

- カードのアプリ名部分のみがリンクになっている

#### 改善提案

- カード全体（`<div>`）を `<Link>` でラップし、どこをクリックしても詳細画面へ遷移できるようにする
- ただし、カード内に「サイトを見る」など別の外部リンクボタンが存在するため、リンクの入れ子にならないよう適切に実装する（例：外部リンクボタンには `e.stopPropagation()` を適用する、またはカード全体ではなくアプリ名のみをリンクにする）

---

### 3-2. 詳細画面へのナビゲーション手段の明確化

#### 現状

- アプリ名がリンクになったことが視覚的にわかりにくい可能性がある

#### 改善提案

- アプリ名にデフォルトでも下線や色（青系）を付与して、リンクであることを明示する
- または、カード右下に「詳細 ›」のようなアイコン付き誘導テキストを小さく配置することで、詳細画面の存在をユーザーに伝える

---

### 3-3. 「サイトを見る」ボタンのアクセシビリティ向上

#### 現状

- 「サイトを見る」ボタンに `target="_blank"` が設定されているが、スクリーンリーダー向けに「新しいタブで開く」旨の補足がない

#### 改善提案

- `aria-label` に「（新しいタブで開く）」を追記するか、ボタン内に視覚的に非表示の補足テキスト（`sr-only`）を追加することで、スクリーンリーダーユーザーへの情報提供を改善する

**変更例**

```tsx
<a
    href={app.siteUrl}
    target="_blank"
    rel="noopener noreferrer"
    aria-label={`${app.name} のサイトを見る（新しいタブで開く）`}
    className="..."
>
    サイトを見る
</a>
```

---

### 3-4. ダークモード切り替えボタンのラベル改善

#### 現状

- ヘッダーのダークモード切り替えボタンのラベルが「ライトモード」「ダークモード」と表示されるが、これは「現在の状態」なのか「切り替え後の状態」なのかが不明瞭

#### 改善提案

- ボタンのラベルを「ダークモードにする」「ライトモードにする」のように切り替え後の状態を示すテキストに変更し、ユーザーがボタンの役割を直感的に理解できるようにする

---

### 3-5. 空の `siteUrl` を持つアプリカードの統一

#### 現状

- `siteUrl` が未設定のアプリでは「サイトを見る」ボタンが表示されないため、カードの高さや見た目がアプリによって異なる可能性がある

#### 改善提案

- `siteUrl` がない場合でも、カードの高さを統一する（例：ボタンエリアに最低高を設定する、または非活性状態のボタンとして表示する）ことで、グリッドレイアウトの視覚的な一貫性を保つ
